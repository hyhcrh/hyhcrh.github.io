闲的没事干

起因

竞赛的机械装置 (圆柱) 需要垂直转动, 使用伺服电机控制转向. 为了使电机运动顺畅, 同时防止装置转到时 碰到离得很近的外侧装置 (圆环柱), 我采用了 H 槽滑轮 + 木板导轨的形式. 为了确定 选定尺寸的滑轮 是否可以在导轨上顺畅运行而不会脱轨 / 卡住, 需要计算导轨在不同宽度下的最小半径.

抽象化问题

滑轮选择如图所示, 半径 20, 内槽宽 12.

导轨外半径不能超过 55, 同时滑轮决定了最大宽度不能超过 12.
木板厚度为 3, 无需考虑.
于是用中学的知识, 得到木板在轮子内部的长度为 $2\sqrt{111}$.

中午告诉了🐯同学方案, 他很感兴趣, 于是加入一起解决问题 =D
接着查看俯视图 (两条横线就是滑轮的两个凸轮缘), 设导轨内半径为 x, 导轨宽度为 y, 得到 $x+y=\sqrt{x^2-111}+12$.

笑死, 当时两个人都把直径半径搞错了 (更正处). 而且直到最后一步我都没发现这个问题

用计算机解方程 (原始)

列不出更多方程了, 🐯同学放弃了解方程.
不过我有 python, 而且相信力大砖飞 (感觉稍加优化后数据量并不大).
打开电脑问了问 DeepSeek, 写了个程序.

写一个 python 程序, 要求定义变量x和y, 满足方程”x+y=\sqrt{x^2-111}+12”, 且 x+y 小于 110. 给定 y, 使用指定的方法算出 x 的最小可能值, 精确到两位小数. 方法: 从 \sqrt{111} 开始, 到 110-y 的所有整数带入 x, 与给定的 y 值算出左右边相差, 选择最小的那个对应的 x 值的整数区间, 步进 0.001 依次带入计算左右边相差, 取最小的输出此时的 x 值.

为了实现这个程序，我们需要按照以下步骤进行：

  1. 定义变量 x 和 y。
  2. 使用给定的 y 值，从 sqrt(111) 开始到 110 - y 的所有整数带入 x，计算方程两边的差值。
  3. 找到差值最小的 x 值的整数区间。
  4. 在这个区间内，以步进 0.001 的方式带入 x，计算方程两边的差值，找到最小的差值对应的 x 值。
  5. 输出 x 的最小可能值，精确到两位小数。

其实之前问了好几遍了, 都用的二分法, 但是得出的结果要么最小, 要么最大. Debug 一下发现这个方程用不了二分法解 (左右相差绝对值会单调增加).
本着 当今计算机性能严重过剩 的信条, 就出现了文中的选择枚举法.
反正只跑一次, 不缺时间.jpg

程序改进

为了保险 (其实就是懒得优化)

  1. (这是真的优化) 因为误差绝对值单调增加, 所以下一个 左右边差值 大于 最小差值 直接 break.
  2. 整数附近 0.001 步进枚举的范围增加了一些, 开始的整数减 2, 结束整数加 2.
  3. 精确枚举时如果到最后 左右边差值 还是大于 0.01 (意味着还需要增加 x), 输出 -1 (满足不了).
  4. for 循环输出当 y 为 10 和 12 步进 0.1 的每一个解, 用 format 做了一个简易表格.
